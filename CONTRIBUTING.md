# Contributing to KIBO

First off, thank you for considering contributing to KIBO! It's people like you that make KIBO such a great AI companion.

## Code of Conduct
By participating in this project, you are expected to uphold a welcoming and inclusive environment. 

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for KIBO. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.
* Make sure you are on the latest version of KIBO.
* Check the issues list to see if the bug has already been reported.
* Provide a clear and descriptive title.
* Describe the exact steps which reproduce the problem.
* Include screenshots if possible!

### Suggesting Enhancements
Enhancement suggestions are highly welcomed! Please outline the feature, the motivation behind it, and roughly how you would implement it.

### Your First Pull Request
1. **Fork** the repo on GitHub.
2. **Clone** the project to your own machine.
3. **Commit** changes to your own branch.
4. **Push** your work back up to your fork.
5. Submit a **Pull Request** so that we can review your changes.

NOTE: Ensure that your code looks clean and adheres to general Python (PEP 8) standards. Run `flake8` and test thoroughly if you add Python logic!

## Development Setup

See the [README.md](README.md) for instructions on setting up your local environment for KIBO.

If you add new custom animations (WebM + PNG fallback), please ensure to place them properly in the `assets/animations/{skin}/action/` directory.

Thank you for contributing!
